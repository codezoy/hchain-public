## 1. 개요

HCHAIN Global Skill에 세 가지 명령어 핸들러를 추가한다.

| 명령어 | 동작 |
|--------|------|
| `/hchain help` | 전체 사용법 출력 |
| `/hchain init` | PROJECT_ROOT에 HCHAIN 구조 생성 |
| `/hchain task` | 직전 요청 또는 인자를 Harness Task로 변환 |

## 2. 경로 변수 정의

SKILL.md 내 모든 경로는 다음 두 변수로 표현한다. 절대경로 하드코딩 금지.

| 변수 | 의미 |
|------|------|
| `PROJECT_ROOT` | 사용자가 현재 작업 중인 프로젝트 루트 디렉토리 |
| `SKILL_ROOT` | HCHAIN Skill 파일이 위치한 디렉토리 |

Claude가 `PROJECT_ROOT`를 결정하는 방법:
1. 현재 세션의 working directory (Claude Code 환경 기준)
2. CLAUDE.md가 존재하는 가장 가까운 상위 디렉토리
3. 불명확 시 사용자에게 확인 요청

## 3. `/hchain help` 설계

### 트리거
- Hard Trigger: `/hchain help`

### Claude 응답 행동
아래 텍스트를 그대로 출력한다. 코드블록 없이 마크다운 형식으로.

```
# HCHAIN Help

HCHAIN는 Claude 채팅 기반 개발 작업 관리 시스템입니다.
요청을 Harness Task로 변환하고 PLAN → RESEARCH → ACTION → REVIEW → VALIDATE → DONE
파이프라인을 통해 안전하게 실행합니다.

## Commands

| 명령어 | 설명 |
|--------|------|
| /hchain help        | 이 도움말 출력 |
| /hchain init        | 현재 프로젝트에 HCHAIN 기본 구조 생성 |
| /hchain task        | 직전 요청 또는 인자를 Harness Task로 변환 |
| /hchain task <text> | <text> 내용 기반으로 Harness Task 생성 |

## Recommended Flow

1. /hchain init        ← 최초 1회 실행
2. 작업 요청 입력
3. /hchain task        ← Harness Task 자동 생성
4. Queue 등록 확인
5. 실행 및 검증
6. 따5코 완료보고

## Workflow Pipeline

PLAN → RESEARCH → ACTION → REVIEW → VALIDATE → DONE

- RESEARCH  : 기술 조사 (Gemini CLI, 읽기 전용)
- ACTION    : 구현 (Claude)
- REVIEW    : 정적 코드 감사 (Codex CLI)
- VALIDATE  : 런타임 검증 (shell 명령어)

## Rules

- 완료보고는 반드시 따5코(`````) 코드박스로 출력
- REVIEW와 VALIDATE는 명시적 승인 없이 생략 불가
- /hchain init 재실행 시 기존 파일 보호 — 누락 항목만 보강
- 모든 경로는 PROJECT_ROOT 기준 (절대경로 하드코딩 금지)
```

## 5. `/hchain task` 설계

### 트리거
- Hard Trigger: `/hchain task`
- Hard Trigger: `/hchain task <임의 텍스트>`

### Case A: 무인자 (`/hchain task`)

```
직전 사용자 메시지 존재 여부 확인:

  존재:
    └─ 해당 메시지를 Goal로 사용
    └─ HCHAIN Skill의 Correct Response Pattern에 따라 Task 생성
    └─ Task ID: TASK_<YYYYMMDD>_<NNN> 형식

  없음 (세션 첫 메시지인 경우):
    └─ 응답:
       "작업 내용을 입력해주세요.
        예시: /hchain task 로그인 기능의 JWT 토큰 갱신 버그를 수정해줘"
```

### Case B: 인자 포함 (`/hchain task <text>`)

```
<text>를 파싱:
  └─ Goal: <text> 전체를 Goal 한 단락으로 사용
  └─ Scope: <text>에서 유추 가능한 포함/제외 항목 도출
  └─ Done Criteria: 검증 가능한 완료 조건 3개 이상 생성
  └─ Steps: PLAN → ACTION → REVIEW → VALIDATE → DONE 표준 단계
  └─ Final Report: 따5코 필수 포함

출력 형식: SKILL.md Correct Response Pattern 준수
```

### Task ID 생성 규칙

```
TASK_<YYYYMMDD>_<NNN>
- YYYYMMDD: 현재 날짜 (오늘 날짜 기준)
- NNN: 001부터 순차 증가
- 같은 날 여러 Task 생성 시 NNN 증가
```
