현재 HCHAIN 문서는 두 가지를 혼동하고 있다:

| 개념 | 올바른 정의 |
|------|-------------|
| **RESEARCH stage** | 워크플로우 단계 — "구현 전 조사" |

RESEARCH stage는 수행 주체를 특정하지 않는다.
RESEARCHER agent는 Gemini CLI를 사용하는 특정 도구다.

## 2. 표준 정의

### 2.1 RESEARCH 단계 표준 정의

```
RESEARCH = 조사 단계 (Investigation Phase)

목적:
  - 구현 전에 필요한 정보를 수집한다
  - 기존 코드/구조/문서를 분석한다
  - 기술적 제약 및 리스크를 파악한다

수행 주체 (수행 가능한 경우 우선 적용):
  - Gemini CLI (RESEARCHER agent) — 외부 기술조사, 웹 검색
  - Claude — 내부 파일 분석, 기존 코드 검토
  - Human — 도메인 지식, 정책 판단

수행 주체는 Task 특성에 따라 결정되며, 특정 Agent에 종속되지 않는다.
```

### 2.2 RESEARCHER 에이전트 (변경 없음)

```
RESEARCHER agent = Gemini CLI 기반 외부 기술조사 도구

  - 호출: gemini headless
  - 권한: 읽기 전용, 외부 문서/표준 검색
  - 출력: JSON (findings[], recommended_architecture, sources[])
  - 사용 시점: RESEARCH 단계 중 외부 기술조사가 필요할 때

RESEARCHER agent는 RESEARCH 단계의 "선택적 도구"이며
RESEARCH 단계 자체를 대표하지 않는다.
```

### 2.3 표준 워크플로우

```
PLAN → RESEARCH → ACTION → REVIEW → VALIDATE → DONE
```

- RESEARCH는 모든 Task에서 명시적으로 수행한다
- SKIP은 "진짜 조사할 내용이 없는 경우"에만 허용
- SKIP 시 반드시 사유를 기록한다

### 2.4 SKIP 정책

```
RESEARCH SKIP 허용 조건:
  - 단순 문구 수정 (오타, 변수명 변경 등)
  - 동일 세션 내 직전 Task에서 이미 조사 완료된 동일 대상
  - 사용자가 명시적으로 조사 불필요를 선언한 경우

SKIP 표기 방법:
  RESEARCH  SKIP  사유: [한 줄 사유]

SKIP 금지:
  - "귀찮아서", "작은 작업이니까" 등 주관적 판단
  - 사유 미기재 SKIP
```
